## What is terraform-aws-wafv2?

`terraform-aws-wafv2` is a Terraform module by [TrussWorks](https://github.com/trussworks) that provisions an AWS WAFv2 Web ACL with sensible defaults and full configurability. It supports both **REGIONAL** (ALB) and **CLOUDFRONT** scopes, and ships with AWS Managed Rule Sets enabled out of the box.

<Info>
  This module requires Terraform `>= 1.0` and the AWS provider `>= 5.0`.
</Info>

## How it works

The module creates an `aws_wafv2_web_acl` resource and optionally:

- Associates it with an ALB via `aws_wafv2_web_acl_association`
- Configures logging via `aws_wafv2_web_acl_logging_configuration`

Rules are evaluated in priority order. The **default action** (allow or block) applies to any request that does not match a rule.

```hcl main.tf
module "wafv2" {
  source  = "trussworks/wafv2/aws"
  version = "~> 3.0"

  name  = "my-web-acl"
  scope = "REGIONAL"

  associate_alb = true
  alb_arn       = aws_lb.main.arn
}
```

## Supported scopes

| Scope | Use case | AWS provider region |
|-------|----------|---------------------|
| `REGIONAL` | ALB, API Gateway, AppSync | Any region |
| `CLOUDFRONT` | CloudFront distributions | Must be `us-east-1` |

<Warning>
  When using `scope = "CLOUDFRONT"`, the AWS provider must be configured with `region = "us-east-1"`. CloudFront WAF resources are global and only available in that region.
</Warning>
