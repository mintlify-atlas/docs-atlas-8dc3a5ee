---
title: "Using WAFv2 with Application Load Balancers"
description: "How to deploy a WAFv2 Web ACL with REGIONAL scope and associate it with an Application Load Balancer."
---

Application Load Balancers (ALBs) are regional resources, so the corresponding Web ACL must use `scope = "REGIONAL"`. The module can create the ALB association automatically when you set `associate_alb = true` and provide the ALB's ARN.

## Basic configuration

Set `scope = "REGIONAL"`, provide the ALB ARN, and enable the association:

```hcl
module "alb_wafv2" {
  source  = "trussworks/wafv2/aws"
  version = "0.0.1"

  name  = "alb-web-acl"
  scope = "REGIONAL"

  alb_arn       = aws_lb.alb.arn
  associate_alb = true
}
```

### Variables used for ALB association

| Variable | Type | Description |
|---|---|---|
| `associate_alb` | `bool` | When `true`, the module creates an `aws_wafv2_web_acl_association` resource linking the Web ACL to the ALB. Defaults to `false`. |
| `alb_arn` | `string` | The ARN of the ALB to associate. Required when `associate_alb = true`. |

<Warning>
  An ALB can only be associated with one Web ACL at a time. Attempting to associate a second Web ACL with an ALB that already has one attached will result in an error.
</Warning>

## Complete example

The following example mirrors the ALB reference configuration from `examples/alb/main.tf`. It combines managed rules, IP set rules, rate-based rules, URL-scoped rate rules, and a custom rule group:

```hcl
module "wafv2" {
  source = "trussworks/wafv2/aws"
  name   = "alb-web-acl"

  scope         = "REGIONAL"
  associate_alb = true
  alb_arn       = aws_lb.alb.arn

  managed_rules = [
    {
      name                 = "AWSManagedRulesAmazonIpReputationList"
      override_action      = "none"
      priority             = 1
      vendor_name          = "AWS"
      rule_action_override = []
    },
    {
      name            = "AWSManagedRulesCommonRuleSet"
      override_action = "none"
      priority        = 2
      vendor_name     = "AWS"
      rule_action_override = [
        {
          name          = "SizeRestrictions_BODY"
          action_to_use = "allow"
        }
      ]
    },
    {
      name                 = "AWSManagedRulesSQLiRuleSet"
      override_action      = "none"
      priority             = 3
      vendor_name          = "AWS"
      rule_action_override = []
    }
  ]

  # filtered_header_rule generates one rule per header_type at (priority + index + 1).
  # With priority = 3 and two header_types, rules are created at priorities 4 and 5.
  filtered_header_rule = {
    header_types  = ["internal-user", "api-client"]
    header_value  = "x-app-client"
    priority      = 3
    action        = "allow"
    search_string = ""
  }

  ip_sets_rule = [
    {
      name       = "monitored-ips"
      priority   = 10
      action     = "count"
      ip_set_arn = aws_wafv2_ip_set.ipset.arn
    },
    {
      name       = "block-all-ips"
      priority   = 20
      action     = "block"
      ip_set_arn = aws_wafv2_ip_set.block_all_ips.arn
    }
  ]

  ip_rate_based_rule = {
    name     = "ip-rate-limit"
    priority = 30
    action   = "block"
    limit    = 100
  }

  ip_rate_url_based_rules = [
    {
      name                  = "ip-rate-foo-limit"
      priority              = 40
      action                = "block"
      limit                 = 100
      search_string         = "/foo/"
      positional_constraint = "STARTS_WITH"
    },
    {
      name                  = "ip-rate-bar-limit"
      priority              = 50
      action                = "block"
      search_string         = "/bar/"
      positional_constraint = "STARTS_WITH"
      limit                 = 200
    }
  ]

  group_rules = [
    {
      name            = aws_wafv2_rule_group.block_countries.name
      arn             = aws_wafv2_rule_group.block_countries.arn
      override_action = "none"
      priority        = 60
    }
  ]
}
```

<Note>
  Rule priorities must be unique within the Web ACL. Lower numbers are evaluated first. The `filtered_header_rule` variable generates one rule per entry in `header_types`, assigning priorities starting at `priority + 1` and incrementing by one per header type — so plan your priority numbering to leave room.
</Note>
