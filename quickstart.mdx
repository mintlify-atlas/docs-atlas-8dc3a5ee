---
title: Quickstart
description: Deploy a WAFv2 Web ACL with Terraform in five steps.
---

This guide walks you through deploying a working WAFv2 Web ACL. The entire process takes about ten minutes.

<Steps>
  <Step title="Prerequisites">
    Before you start, confirm the following are in place:

    - **Terraform >= 1.0** installed and on your `PATH`.
    - **AWS credentials** configured — either via environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`), an AWS profile, or an IAM role.
    - **AWS provider >= 5.0** declared in your root module.

    Add the provider to your root module if it is not already present:

    ```hcl provider.tf
    terraform {
      required_version = ">= 1.0"

      required_providers {
        aws = {
          source  = "hashicorp/aws"
          version = ">= 5.0"
        }
      }
    }

    provider "aws" {
      region = "us-east-2"
    }
    ```
  </Step>

  <Step title="Choose your scope and write the module block">
    The module supports two scopes. Pick the one that matches the resource you are protecting.

    <CodeGroup>

    ```hcl REGIONAL (ALB)
    module "alb_wafv2" {
      source  = "trussworks/wafv2/aws"
      version = "0.0.1"

      name  = "alb-web-acl"
      scope = "REGIONAL"

      alb_arn       = aws_lb.alb.arn
      associate_alb = true
    }
    ```

    ```hcl CLOUDFRONT
    module "cloudfront_wafv2" {
      source  = "trussworks/wafv2/aws"
      version = "0.0.1"

      name  = "cloudfront-web-acl"
      scope = "CLOUDFRONT"
    }
    ```

    </CodeGroup>

    <Note>
      Both examples deploy the module with all defaults. This enables 6 AWS Managed Rule Sets and sets `default_action = "allow"` for unmatched requests. No additional variables are required to get a functional WAF.
    </Note>
  </Step>

  <Step title="Configure the provider for CloudFront scope">
    Skip this step if you are using `REGIONAL` scope.

    AWS requires `CLOUDFRONT`-scoped Web ACLs to exist in `us-east-1`. Use a provider alias so the WAF resource is created in the correct region while the rest of your infrastructure can remain in any region.

    ```hcl provider.tf
    provider "aws" {
      region = "us-east-2"  # your primary region
    }

    provider "aws" {
      alias  = "us_east_1"
      region = "us-east-1"
    }
    ```

    Pass the aliased provider to the module:

    ```hcl main.tf
    module "cloudfront_wafv2" {
      source  = "trussworks/wafv2/aws"
      version = "0.0.1"

      providers = {
        aws = aws.us_east_1
      }

      name  = "cloudfront-web-acl"
      scope = "CLOUDFRONT"
    }
    ```

    Then reference the output ARN in your CloudFront distribution:

    ```hcl cloudfront.tf
    resource "aws_cloudfront_distribution" "main" {
      # ... other configuration ...

      web_acl_id = module.cloudfront_wafv2.web_acl_id
    }
    ```
  </Step>

  <Step title="Apply the configuration">
    Run the standard Terraform workflow from your module directory.

    ```bash
    terraform init
    ```

    Review the planned changes before applying:

    ```bash
    terraform plan
    ```

    The plan output will show one `aws_wafv2_web_acl` resource being created, plus `aws_wafv2_web_acl_association` if you set `associate_alb = true`.

    Apply when you are satisfied with the plan:

    ```bash
    terraform apply
    ```

    Type `yes` when prompted to confirm. The Web ACL is typically provisioned within 30 seconds.
  </Step>

  <Step title="Check the output">
    After a successful apply, retrieve the Web ACL ARN from the `web_acl_id` output:

    ```bash
    terraform output web_acl_id
    ```

    Example output:

    ```
    "arn:aws:wafv2:us-east-2:123456789012:regional/webacl/alb-web-acl/a1b2c3d4-1234-5678-abcd-ef0123456789"
    ```

    You can use this ARN to:

    - Reference the Web ACL from other Terraform resources (e.g., a CloudFront distribution via `web_acl_id`).
    - Verify the Web ACL exists in the [AWS WAF console](https://console.aws.amazon.com/wafv2/homev2).
    - Associate the ACL with additional resources outside of Terraform.

    <Note>
      The output is named `web_acl_id` for historical compatibility, but its value is the full ARN, not a short ID.
    </Note>
  </Step>
</Steps>
